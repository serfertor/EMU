🏗️  РЕАЛИЗОВАННАЯ АРХИТЕКТУРА
═══════════════════════════════════════════════════════════════════════════════

КОМПОНЕНТЫ ПРОЦЕССОРА:
├─ 4 регистра общего назначения (EAX, EBX, ECX, EDX)
├─ Счётчик команд (PC)
├─ Регистр команд (IR) с отображением в ассемблере и двоичном коде
├─ Регистр флагов (Z - Zero, S - Sign)
└─ Память данных (4096 ячеек)

СИСТЕМА КОМАНД (16 операций):
├─ Передача:    MOV
├─ Арифметика:  ADD, SUB, MUL, DIV
├─ Логика:      AND, OR, XOR, NOT
├─ Специальные: INC, CMP
└─ Управление:  JMP, JZ, JNZ, JS, JNS

ТИПЫ АДРЕСАЦИИ (4 варианта):
├─ 00 Регистровая       (eax, ebx, ecx, edx)
├─ 01 Непосредственная  (123, -45)
├─ 10 Косвенно-регистровая ([eax], [ecx])
└─ 11 Прямая            ([0], [100])

АРХИТЕКТУРА:
├─ Гарвардская (отдельная память программ и данных)
├─ 3-адресные команды
└─ 40-битные инструкции [opcode(4)|ad1(2).op1(16)|ad2(2).op2(16)]

📝 ПРИМЕРЫ ПРОГРАММ
═══════════════════════════════════════════════════════════════════════════════

ПРИМЕР 1: Простое сложение
    mov eax 5
    add eax 3
    
    Результат: EAX = 8

ПРИМЕР 2: Цикл (сумма от 1 до 5)
    mov eax 0
    mov ecx 5
    add eax ecx
    sub ecx 1
    jnz 2
    
    Результат: EAX = 15

ПРИМЕР 3: Работа с памятью
    mov [0] 42
    mov eax [0]
    add eax 8
    
    Результат: EAX = 50

ЗАДАЧА 1: Сумма элементов массива
    Дано: 6 элементов в памяти
    Найти: их сумму
    Ожидаемый результат: 120
    
    Можно загрузить кнопкой "Задача 1: Сумма"

ЗАДАЧА 2: Свертка двух массивов
    Дано: два массива по 10 элементов (1 и 2)
    Найти: скалярное произведение
    Ожидаемый результат: 20
    
    Можно загрузить кнопкой "Задача 2: Свертка"

🎮 ИНТЕРФЕЙС ПОЛЬЗОВАТЕЛЯ
═══════════════════════════════════════════════════════════════════════════════

ГЛАВНОЕ ОКНО разделено на 4 части:

1. ВЕРХНЯЯ ЧАСТЬ - Ввод программы
   └─ Большое текстовое поле для ввода кода на ассемблере

   ├─ EAX, EBX, ECX, EDX (текущие значения)
   ├─ PC (программный счётчик)
   ├─ IR (текущая инструкция)
   ├─ Bin (двоичное представление)
   └─ Z, S флаги

3. СРЕДНЯЯ КОЛОНКА - Память и статус
   ├─ Содержимое памяти (первые 30 ячеек)
   └─ Статус выполнения и результат

4. ПРАВАЯ КОЛОНКА - Консоль
   └─ Лог всех выполненных команд

🆘 РЕШЕНИЕ ПРОБЛЕМ
═══════════════════════════════════════════════════════════════════════════════

ПРОБЛЕМА: ModuleNotFoundError: No module named 'tkinter'
РЕШЕНИЕ:
  Linux (Debian): sudo apt-get install python3-tk
  Linux (Fedora): sudo dnf install python3-tkinter
  macOS: python3 -m tkinter (проверить установку)
  Windows: Переустановить Python, выбрав "tcl/tk"

ПРОБЛЕМА: Программа зависает
РЕШЕНИЕ:
  └─ Вероятно бесконечный цикл
  └─ Нажмите кнопку "Сброс"
  └─ Используйте "Шаг" для пошагового выполнения

ПРОБЛЕМА: Неправильный результат
РЕШЕНИЕ:
  └─ Используйте пошаговое выполнение (кнопка "Шаг")
  └─ Проверьте адреса памяти
  └─ Смотрите консоль для лога команд

ПРОБЛЕМА: Ошибка "Неизвестная команда"
РЕШЕНИЕ:
  └─ Проверьте правильность написания команды
  └─ Команды чувствительны к регистру (add, не ADD)
  └─ Смотрите примеры в QUICKSTART.md
